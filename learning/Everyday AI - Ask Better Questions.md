## Day 2: Ask Better Questions

### What you’ll learn today 

You will learn a simple prompting structure that produces more precise, more accurate results from any AI tool:

_**Goal → Context → Constraints → Format**_

This structure makes it easier for AI to understand what you want, why you want it, and how the output should look.

Once you learn it, your writing, planning, and thinking tasks become faster and smoother.

### Tool for today: Claude (free version)

Claude is a family of large language models (LLMs) developed by Anthropic. It is designed as an AI assistant capable of tasks like writing, coding, analysis, and conversation.

### Today’s challenge: Improve a weak prompt

**Step 1: Choose a real task**

Pick something from your actual day that requires writing or clarity, such as:

- An email you need to write
- Instructions you need to give
- A message you need to clean up
- A short paragraph you want polished

This real-world context will make the result more meaningful.

**Step 2: Write the weak version of your prompt**

Type your request the way you would normally ask it.

- **Example**: “_Help me write an email about a project update._”

(This is intentionally vague.)

![Screenshot of weak prompt in Claude console](https://media.licdn.com/dms/image/v2/D4E0DAQGwLCwbQi65QA/learning-article-inline-scale_1000_2000/B4EZq3xVi_JgAo-/0/1764019755582?e=1774429200&v=beta&t=p_kurFKQbj4rs4e21fFjdizghl45kkDFyDRMm2xw6vY)

Claude asks for more detail because the prompt is vague. It often takes a few back-and-forth exchanges to shape a good prompt. But if you got it right on the first try, you would save time, avoid unnecessary steps, and move through your work much faster.

![Screenshot of Claude’s response to a weak prompt](https://media.licdn.com/dms/image/v2/D4E0DAQFqnC8JYJeqqQ/learning-article-inline-scale_1000_2000/B4EZq3xefRKMAU-/0/1764019793658?e=1774429200&v=beta&t=51g1_PdxruST0JSqgX3-tpULbQ7k8HZOHuETAYPmKbI)

**Step 3: Transform the prompt using the four-part structure**

Start a new chat:

![Screenshot of starting a new chat in Claude](https://media.licdn.com/dms/image/v2/D4E0DAQFz983anLivdg/learning-article-inline-scale_1000_2000/B4EZq3xnRNGYAU-/0/1764019828182?e=1774429200&v=beta&t=4fpT55L0AceWYgE0Nf1tENbTL5S8uz1BtVbM9ctDNeI)

Now rewrite your vague prompt using:

- **Goal**: What you want
- **Context**: What background the AI needs
- **Constraints**: What boundaries to follow
- **Format**: How the output should look

Example structure:

- **Goal**: Write a professional project update email
- **Context**: The timeline shifted, and the team needs clarity
- **Constraints**: Keep it concise, neutral in tone, and action-oriented
- **Format**: Provide a subject line and full email

Don’t forget to add the supporting information that Claude needs to complete the task, in this example, a draft of the email that needs to be improved.

Paste the updated prompt and supporting information into Claude.

Here is my example:

_Goal: Help me rewrite an email so it is clear, professional, and easy to understand._

_Context: This is a draft I plan to send, but it is messy and unclear. I want to improve the wording, flow, and overall readability while keeping the original intent._

_Constraints: Keep the tone neutral and professional. Make the message concise. Do not add new information that is not already in the draft._

_Format: Provide:_

- _A revised version of the email_
- _A short explanation of the changes you made_

_Draft Email:_ 

_"Hi team, I wanted to check in about the report. I know things have been busy and I’m behind on getting my part over. I think I can finish it by sometime next week but I wanted to see if anyone already started on the edits or if I should just keep going. Let me know."_

![Screenshot of updated prompt](https://media.licdn.com/dms/image/v2/D4E0DAQGUf4mvmGMr5g/learning-article-inline-scale_1000_2000/B4EZq3yNh9HgAY-/0/1764019984982?e=1774429200&v=beta&t=uc2enYNnjIooqi3wlQIpqdMjzbCyOncYH173ceAwo1M)

This time, Claude got right to work, no back and forth needed.

![Claude updated email](https://media.licdn.com/dms/image/v2/D4E0DAQE6qHWhIO1H8g/learning-article-inline-scale_1000_2000/B4EZq3yXf3GUAU-/0/1764020025781?e=1774429200&v=beta&t=mm39hBXLKV8zcWyZT-7omuBF7Pus83lUZQUaMb4JLH8)

**Step 4: Compare the results**

Look at the difference in clarity, structure, and usefulness.

Ask yourself:

- Did the structured version feel more accurate?
- Did it save me time?
- Would I use this method again?

This is where improvements become obvious.
